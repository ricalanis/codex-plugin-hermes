# Decisions

- 2026-07-19: Mirror **semantics** of codex-plugin-cc, not implementation (per PRD non-goals). Core wrapper is Bash (`scripts/codex-companion.sh`) around `codex exec --json`, not a Node app-server broker.
- 2026-07-19: Orchestration split — Claude (frontier) plans/verifies; Codex executes bounded implementation tasks via the codex-rescue channel; verification is contract-based (checklists/tests authored before delegation).
- 2026-07-19: **Upstream mirrors are no longer committed.** `ref/` was carrying a vendored 608K copy of `openai/codex-plugin-cc` (64 of 66 tracked files). Two reasons to un-vendor before going public: we shouldn't redistribute upstream code when it's already public and Apache-2.0-attributed in our NOTICE, and — more importantly — the vendored copies had no `.git`, so the drift loop could never actually pull. Mirrors are now cloned on demand via `check-drift.sh --bootstrap`, which makes them real clones and fixes the loop. `ref/README.md` stays committed to document the bootstrap.
- 2026-07-19: **`check-drift.sh` must distinguish "no drift" from "cannot check."** It previously `continue`d past mirrors that weren't git clones and then printed `no drift` — a false green over a loop that was doing nothing. It now exits 2 with `CANNOT CHECK` when nothing was verified. Same principle as the acceptance suite's skipped-check reporting: silence about what wasn't checked is worse than a failure.
- 2026-07-19: Project-wide autonomy delivered as repo skills (`.claude/skills/` + Hermes-facing `skills/`) implementing the pull-reflect drift loop described in PRD.md.
- 2026-07-19: **History rewritten to remove the vendored upstream before public release.** The scaffold commit carried all 608K of `openai/codex-plugin-cc`; removing it from HEAD left it in history, which a public repo exposes. Rewrote all 3 commits with `git filter-branch --index-filter`, purged `refs/original`, expired reflogs, and `gc --prune=now` (`.git` 1.1M → 184K), then force-pushed with a `--force-with-lease` pinned to the verified remote SHA. A `git bundle` backup was taken first. Verified from a fresh public clone that no `ref/codex-plugin-cc` path and no upstream source identifiers survive in any commit. Legally the vendoring was fine (Apache 2.0 + NOTICE); this is about not shipping someone else's tree in ours.
