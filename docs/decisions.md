# Decisions

- 2026-07-19: Mirror **semantics** of codex-plugin-cc, not implementation (per PRD non-goals). Core wrapper is Bash (`scripts/codex-companion.sh`) around `codex exec --json`, not a Node app-server broker.
- 2026-07-19: Orchestration split — Claude (frontier) plans/verifies; Codex executes bounded implementation tasks via the codex-rescue channel; verification is contract-based (checklists/tests authored before delegation).
- 2026-07-19: **Upstream mirrors are no longer committed.** `ref/` was carrying a vendored 608K copy of `openai/codex-plugin-cc` (64 of 66 tracked files). Two reasons to un-vendor before going public: we shouldn't redistribute upstream code when it's already public and Apache-2.0-attributed in our NOTICE, and — more importantly — the vendored copies had no `.git`, so the drift loop could never actually pull. Mirrors are now cloned on demand via `check-drift.sh --bootstrap`, which makes them real clones and fixes the loop. `ref/README.md` stays committed to document the bootstrap.
- 2026-07-19: **`check-drift.sh` must distinguish "no drift" from "cannot check."** It previously `continue`d past mirrors that weren't git clones and then printed `no drift` — a false green over a loop that was doing nothing. It now exits 2 with `CANNOT CHECK` when nothing was verified. Same principle as the acceptance suite's skipped-check reporting: silence about what wasn't checked is worse than a failure.
- 2026-07-19: Project-wide autonomy delivered as repo skills (`.claude/skills/` + Hermes-facing `skills/`) implementing the pull-reflect drift loop described in PRD.md.
- 2026-07-19: **History rewritten to remove the vendored upstream before public release.** The scaffold commit carried all 608K of `openai/codex-plugin-cc`; removing it from HEAD left it in history, which a public repo exposes. Rewrote all 3 commits with `git filter-branch --index-filter`, purged `refs/original`, expired reflogs, and `gc --prune=now` (`.git` 1.1M → 184K), then force-pushed with a `--force-with-lease` pinned to the verified remote SHA. A `git bundle` backup was taken first. Verified from a fresh public clone that no `ref/codex-plugin-cc` path and no upstream source identifiers survive in any commit. Legally the vendoring was fine (Apache 2.0 + NOTICE); this is about not shipping someone else's tree in ours.
- 2026-07-19: **Both upstream repos are bootstrapped as real clones; no hand-written API reference.** A summary doc describing Hermes' API was drafted and rejected: a maintenance loop whose Hermes knowledge lives in prose drifts silently from the platform it targets, which is the exact failure mode the pull-reflect loop exists to prevent. `ref/hermes-agent` is now cloned the same way as `ref/codex-plugin-cc` (`check-drift.sh --bootstrap`), giving maintainers the real source — including the `tools/` package the earlier slimmed copy lacked. Drift in hermes-agent is now first-class: a changed hook signature can break this plugin even when upstream codex-plugin-cc never moves. `docs/research/*` remain as navigational maps, explicitly subordinate to the source.
- 2026-07-19: **Rejected a proposed `changed_paths` short-circuit in the stop gate.** The idea was to skip a Codex turn when `pre_verify` reports no changed files. Reading `agent/verify_hooks.py` in the hermes-agent mirror shows the hook only fires "when the agent has edited code and is about to verify/finish" — Hermes already guarantees the condition the optimization would re-check, so it would add a branch that never fires. Also noted: `agent.max_verify_nudges` (default 3) bounds consecutive continue directives per turn, so a misbehaving gate cannot loop a session forever. Both facts came from reading source that a prose summary would not have carried — see the decision above about bootstrapping the real mirror.
- 2026-07-19: **Drift detection moved to a committed `upstream.lock`; it no longer needs the mirrors.** The previous detector compared a local clone's HEAD before and after a `git pull`. That cannot work in an ephemeral container (Codex Web, CI): `ref/` is gitignored, so every run cloned fresh, had no "before", and would have reported `no drift` every week forever — a check that could never fail. `upstream.lock` records the commits this repo was reconciled against; the detector compares them to live `git ls-remote` HEADs, so it works with an empty `ref/` and costs no clone. `--bootstrap` now only clones when you actually need to read changed source, and `--update-lock` closes the loop after triage. Acceptance covers all three states, including that a missing lockfile yields CANNOT CHECK rather than a false all-clear.
