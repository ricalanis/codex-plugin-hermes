# Known Issues / Lessons

- `~/.hermes/plugins/` is empty on this machine — no installed-plugin example to copy. Plugin manifest format must come from hermes-agent source (ref/hermes-agent), not from a local example.
- Hermes hooks are declared in `~/.hermes/config.yaml` under `hooks:` (event → list of `{command, matcher, timeout}`), NOT in a per-plugin hooks.json like Claude Code. First use requires consent via `~/.hermes/shell-hooks-allowlist.json` (`hermes hooks` manages this). A plugin that ships hooks must document/install them into config.yaml (or rely on plugin-manifest hook support if it exists).
- `ref/` is read-only by contract (PRD "Mirror pattern"). Never edit files there.
- **2026-07-19 — buggy acceptance check blamed the implementer.** The SKILL.md frontmatter check in `tests/acceptance.sh` used `re.S` (DOTALL) with `(.+)$`, so the inline `description:` capture greedily swallowed the rest of the frontmatter (146 chars instead of 52) and failed a perfectly valid skill. **Cause:** DOTALL plus a greedy quantifier anchored with `$` under `re.M`. **How to avoid:** when a contract check fails, first reproduce the check in isolation against a known-good input before reporting a defect to Codex; and after fixing a check, prove it still fails on a deliberately bad input (a check that only passes is worthless). Verification code deserves the same scrutiny as implementation code.
- **2026-07-19 — two phantom defects created by the test method itself.** (a) Smoke-testing `scripts/codex-companion.sh` while Codex was still editing it produced a syntax error at a line that was fine moments later — bash reads scripts incrementally, so a concurrent write looks like a real parse bug. (b) Testing the no-`setsid` fallback by copying the script to `/tmp/ns.sh` made every background job hang in `queued`; the cause was `$SELF` resolving to the script's hardcoded *basename* (`/tmp/codex-companion.sh`, nonexistent), not a defect in the fallback — which works correctly when the file keeps its real name. **How to avoid:** never rename or relocate a script under test when it self-references, and never smoke-test a file another agent is actively writing. Both times the evidence looked damning and both times the implementation was innocent — reproduce cleanly before reporting.
- **2026-07-19 — a "green" drift check that verified nothing.** `check-drift.sh` skipped mirrors lacking `.git` and then printed `no drift`, so a loop that never ran looked healthy. Both mirrors were plain directories, so it had *always* been a no-op. **Cause:** treating "couldn't check" as "nothing to report". **How to avoid:** any checker needs three outcomes — pass, fail, and *could not verify* — and the third must be as loud as a failure. This is the same class of bug as the silently-skipped shellcheck gate; when a check can be skipped, make the skip part of the output.
- `hermes plugins install` accepts only a **Git URL or `owner/repo` shorthand** — not a local filesystem path. Local development installs must symlink into `~/.hermes/plugins/<name>` instead.
- **2026-07-19 — a dev symlink broke the real install, and looked like a Hermes bug.** After local-dev testing left `~/.hermes/plugins/codex -> /path/to/checkout`, `hermes plugins install` failed with *"Invalid plugin name 'codex': resolves outside the plugins directory."* **Cause:** `plugins_cmd.py` computes `target = (plugins_dir / name).resolve()`, and `.resolve()` follows symlinks — so the destination resolved to the checkout, outside `~/.hermes/plugins`, and the path-traversal guard correctly rejected it. The manifest name was never the problem. **How to avoid:** remove a dev symlink before switching to a git install; and when an error names *your* identifier, check for local state shadowing it before concluding the tool is broken. Documented in README under Install.
- **2026-07-19 — I reintroduced the silent-skip bug I had just fixed.** After adding the `SKIPPED` mechanism for the shellcheck gate, two later conditional checks (`shellcheck doctor.sh`, and the check-drift shellcheck) were written as bare `if command -v ...; then check ...; fi` with no `else`, so they vanished from the report entirely. The first weekly run surfaced it: the container reported 52 passed with only two skips named, when four checks had actually been skipped. **Cause:** the skip-reporting was a one-off inline branch rather than a reusable primitive, so every new conditional check had to remember to opt in. **Fix:** added a `check_needs <tool> <desc> <cmd...>` helper; conditional checks now route through it and cannot skip silently. **Lesson:** when a class of bug is worth preventing, encode it in a primitive — a convention you have to remember is one you will forget.
- **2026-07-19 — published a comparison that omitted the closest competitor.** The README's "How this differs" section compared the plugin only against `/codex-runtime` (an inference-backend swap) and concluded the plugin uniquely offered delegation, background jobs, and review. Hermes in fact **bundles a `codex` skill** (`skills/autonomous-ai-agents/codex`) that already does `codex exec` delegation, `background=true` with `process` poll/log/kill, and `codex review --base` — so two of those claims were false. **Cause:** I compared against the integration I happened to know about instead of enumerating all of them; earlier research had covered the plugin/hook/command systems and never swept `skills/`. **How to avoid:** before claiming a capability is novel, grep the whole tree for the capability, not just the subsystem you were working in — and when writing a comparison, look hardest for the thing that makes your work redundant.
